## Prácticas recomendadas

### Java (Clean Code)

- Usar nombres de clases y métodos descriptivos.
- Mantener una clase con una responsabilidad clara.
- Evitar métodos largos y duplicación de código.
- Mantener consistencia en formato e indentación.
- Separar lógica de negocio de entrada/salida cuando sea posible.
